# AR Location Memo App
### 物理的な場所とデジタル情報を紐付ける、QRコードを用いたAR管理システム

[動作デモ]

## 1. プロジェクト概要
本アプリは、現実世界の「場所」や「モノ」に対して、QRコードを介してデジタル情報を付与・管理できるARシステムです。
単なるラベルシールの代替ではなく、省スペースでの情報集約、プライバシー保護、スキャン履歴のデータ化など、デジタルならではの付加価値を提供することを目的に開発しました。

### 解決する課題
- **情報量の制限**: 物理ラベルでは書ききれない詳細なメモやリストを1枚のQRに集約。
- **視覚的ノイズの削減**: メモ内容を隠蔽し、スキャン時のみ表示することで空間の美観を維持。
- **管理の効率化**: 「デジタルで作成・蓄積」→「まとめて印刷・貼付」というワークフローにより、整理整頓をシステム化。

## 3. 技術的な工夫と解決した課題

### 3.1 実用性を重視したARトラッキング方式の転換
* **課題**: 当初はGPSや平面検知（Plane Detection）による配置を検討していましたが、屋内での精度不足や、特徴の少ない壁面での位置ズレが実用上の障壁となりました。
* **解決**: 確実な自己位置推定が可能な「QRコードを起点とするイメージトラッキング方式」を採用。場所の特定精度を100%に高めることで、実用的な管理ツールとしての信頼性を担保しました。

### 3.2 ネイティブ統合によるスキャン体験の最適化
* **課題**: Unity標準のカメラキャプチャ経由のスキャンでは、低照度下や斜めからの角度における認識率が低く、ユーザー体験を損なっていました。
* **解決**: Androidネイティブの強力なスキャンエンジンである「Google ML Kit」をJNI経由で組み込みました。結果、フォーカス速度と認識精度が大幅に向上し、ストレスのない「かざした瞬間の反応」を実現しました。

### 3.3 印刷用データの生成スピード改善
* **課題**: A4用紙のサイズに合わせて高解像度の印刷データを作成する際、処理が非常に重く、ボタンを押してから共有画面が開くまで数秒間待たされるストレスがありました。
* **解決**: 一度にすべての画像を処理するのではなく、メモリを効率的に使う「非同期処理」と「動的なレイアウト生成」を組み合わせる工夫をしました。これにより、待ち時間を大幅に短縮し、ストレスなくスムーズに印刷・共有ができる快適な操作感を実現しました。

---

## 4. 設計において意識したポイント
* **役割の分離**: 画面表示（UI）とデータの処理（ロジック）を切り離して設計することで、不具合の特定を容易にし、コードの管理をしやすくしました。
* **スムーズな操作感**: 読み取りなどの重い処理が画面の動きを妨げないよう、非同期的な処理を意識して実装しました。
